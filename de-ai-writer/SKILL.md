# De-AI Writer Skill

接收一篇 Markdown 文档，按以下规则逐项检查并改写，输出去掉 AI 味的新 Markdown 文档。

## 输入

用户提供的 Markdown 文档全文（含 frontmatter）。

## 输出要求

- 直接给出完整改写后的 Markdown 文档
- 保留原文 frontmatter（title、author、date、cover、tags、summary 等字段）不改动
- 不解释修改过程，不展示检查步骤，不附加任何说明

---

## 改写规则（按优先级依次执行）

### 1. 风格与语气

- 表达专业，语气稳，不端着；确实有助于理解时，可少量加入轻微幽默
- 靠具体信息、有效判断、清晰推理支撑内容，不靠夸张语气和"气势词"
- 不为"显得高级"而堆修辞，不用过度煽动或过度抒情的表达

### 2. 叙述视角一致

全文只选一种主视角，保持一致：
- "我们"视角
- "你"视角
- 中性说明视角（如"本文""该方法""这类问题"）

段落间不频繁切换主语立场；如需切换（例如从说明转为建议），要有明确过渡。

### 3. 信息密度与表达方式

- 优先写结论、依据、条件、做法，不绕弯子
- 删掉空转句：不写不增加信息的开场、过渡、总结
- 不用套路化自问自答（如"为什么会这样？其实很简单……"）
- 每句话至少承担一种作用：提供信息 / 推进推理 / 限定条件 / 补充例外 / 解释因果

### 4. 中文句式与节奏

- 长短句搭配，避免整段都是一种句长
- 不连续使用同构句（如连续三句都用"首先/其次/最后"或"我们可以……"）
- 排比只在确有信息增量时使用，不为制造气势而排比
- 句子结构贴近中文自然表达，不照搬英文逻辑顺序

### 5. 标点与格式习惯

- 不用长破折号（——）制造节奏或强调
- 不用连续感叹号、省略号（如"！！！""……"）制造情绪
- 不滥用括号插入解释，导致主句被打断
- 并列项优先用清晰分句，不靠标点堆砌层次
- 术语、英文、数字的格式在全篇保持一致（不要一会儿全角一会儿半角）
- **绝对不要在文章中添加多余的英文备注**（如“中文（English）”式的中英双语解释）。直接使用顺畅的中文表达，必要时只保留原生英文专有名词本身即可，坚决去掉各种画蛇添足的英文备注！

### 6. 词汇约束

#### 6.1 禁用空泛套话（及同类变体）

以下表达及其变体一律删改：
- 在当今快速变化的时代……
- 随着XX的不断发展……
- 面对日益复杂的环境……
- 不难发现……
- 值得注意的是……（后面如果没有新信息）
- 毋庸置疑……
- 综上所述……（如果只是重复前文）
- 从某种意义上说……
- 可以说是/可谓是（若后面没有具体判断）

#### 6.2 禁用套路句式（及同类改写）

以下结构一律删改：
- "这不仅仅是X，更是Y……"
- "X不仅仅是Y，它还是Z……"
- "这正是X发挥作用的地方……"
- "真正的问题在于……（但后文并未展开）"
- "本质上/归根结底/说到底……"（如果只是换个说法重复）

#### 6.3 用词偏好

- 优先使用朴素、具体、可验证的词
- 少用"高大上但空"的词：赋能、抓手、闭环、打法、矩阵、链路（若未定义）；全面提升、显著优化、有效支撑（若无指标或机制）
- 能写具体动作，就不写抽象名词：
  - "推进协同" → "统一规则口径并明确交接人"
  - "提升效率" → "把审批时间从2天降到4小时"

#### 6.4 形容词限制

形容词只在能增加具体信息时保留：
- 可保留：长期/短期（最好量化）、小范围/大批量（最好补数字）、高延迟/低吞吐、严格/有限
- 删除纯评价型形容词：优秀、强大、出色、先进、创新（若无证据支撑）

#### 6.5 类比限制

- 类比少用
- 不以"你可以想象……"开头
- 类比必须帮助理解机制或边界，不能只负责"好懂"但牺牲准确性

### 7. 语感检查（重点排查翻译腔和模板腔）

逐一检查并修改：
- **主语重复**：连续多句都以"我们/它/这"开头，读起来机械
- **名词化过度**：把动作全写成抽象名词（如"进行优化实施推进"），缺少明确动作
- **逻辑连接词过密**：因此、此外、同时、然而、总之连续出现，像拼接段落
- **句子太满**：一个句子塞入太多定语、括号、并列项，读起来费力
- **语义重复**：同一意思换三种说法重复表达
- **段首模板化**：每段都用"首先/其次/再次/最后"开头

### 8. 准确性与确定性

- 忠实于原文背景、数据和来源，不自行补设定
- 不夸大结论，不把推测写成事实
- 避免过于绝对的表达（例如“绝不是”、“肯定”等），保持客观严谨
- 不确定内容：省略，或明确标注条件，或标注"目前无法确认/取决于X"

### 9. 结构检查

- 段落开头就给出该段核心意思，不绕
- 段落内部顺序：结论 → 原因/依据 → 限制条件（或例外）
- 相邻段落之间有自然承接，不硬切话题
- 列表项尽量同一维度，不混写原则、动作、例子、情绪判断

---

## 执行方式

当用户提供 Markdown 文档时：

1. 读取完整文档内容
2. 按上述9条规则依次扫描并改写
3. 保留 frontmatter 原样不动
4. 直接输出改写后的完整 Markdown 文档
5. 不附加任何说明、解释或改写日志

> [!TIP]
> **KK's Tip**: 去 AI 味的第一步是“去油”。把那些宏大的连接词删掉，让逻辑自然流淌。删减是为了更好的表达，每一处留白都是给读者思考的空间。
